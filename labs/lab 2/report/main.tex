To complete this lab a simple website was built using `Node.js` as the backend. All of the testing was performed using OS X and node version \code{v8.6.0}. All information was stored securely within a postgresql database.\\

To perform the text messaging we chose to use Twilio. Twilio offers a service trough which you can send text massages from within your application. One assumption which has been made is that the action of sending the text message is secure. To further mimic the protocol of real world 2FA we had the text message contain a secure random 6 digit code which is unique to the user's session and expires after a few minutes. Many web applications, including sites like Google, use this form of two factor authentication.\\

Two factor authentication is important because it offers a second factor of security for users ensuring that hackers can't get into their account as easily. To sign in with 2FA you are now required to have physical access to the person's cellphone. 2FA also allows the website to further verify that the user is who they say they are.\\\\

The design made use of BCrypt to securely store passwords within the database. When a users signs up the password is immediately hashed using 10 iterations and then removed from memory to ensure the plaintext password should never be leaked.\\

Further more our application made use of SSL to ensure that the connection between the user and the application is encrypted. When a user connects to the server, the server sends a public key. The client then uses that public key to to encrypt a generated symmetric key which is then sent back to the server. The server is able to use its private key to decrypt the symmetric key and perform secure communications using that key.\\

These security measures, combined with the 2FA, provide the server a way to validate the user is who they say they are while offering peace of mind to the user that they are connected to an application designed to keep their information and communications safe.
